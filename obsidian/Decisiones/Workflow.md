# Workflow

## Division de trabajo

| Quien        | Que                                                          |
| ------------ | ------------------------------------------------------------ |
| Codex/Claude | Codigo Godot, refactors, smoke tests, documentacion tecnica. |
| Tutupa       | Playtest, tuning, diseno, Blender, aprobacion de feel.       |

## Ritual de cierre

1. Verificar headless si hubo codigo.
2. Jugar si hubo feel.
3. Actualizar estado E0-E4.
4. Actualizar nota afectada de esta boveda.
5. Commit.

## Reglas anti-muerte

- Una tarea activa a la vez.
- Si aparecen tres subtareas, se anotan y se vuelve a la activa.
- Tres semanas sin tocar el proyecto: primera sesion es solo jugar el build.
- Ideas nuevas fuera de H1 van a [[Roadmap Futuro]].
